# API Key Authentication — Implementation Plan

## Context

The REST API (`devmemory serve`) is currently wide open: zero authentication, CORS allows all origins. This is fine for localhost-only use, but the server can be exposed to remote machines (CI/CD webhooks, VSCode extension, GitHub Actions, cross-machine agents). Without auth, any process that can reach port 7711 can read all memories, write new ones, or invoke the webhook ingest endpoint.

Goal: add optional API key authentication to the REST API that is:
- **Off by default** (no key in config → server works as today, localhost-safe)
- **On when a key is configured** (all endpoints require `Authorization: Bearer <key>`)
- **Easy to use** — one CLI command to generate a key, one header to use it

---

## Design

### Key storage
Add `[api]` section to `~/.config/devmemory/config.toml`:
```toml
[api]
key = "a3f9c12d..."   # 64-char hex, generated by devmemory api-key generate
```
Uses the existing `core/config.py` `load()`/`save()` pattern — no new dependencies.

### Auth behaviour
- If `[api] key` is **absent** → server accepts all requests (current behaviour, safe for localhost).
- If `[api] key` is **present** → all routes require `Authorization: Bearer <key>`. Missing or wrong key returns `401 Unauthorized`.
- `devmemory serve --no-auth` flag overrides and disables enforcement even if a key is configured (useful for debugging or trusted LAN).

### Header format
`Authorization: Bearer <key>` — standard, works with curl, GitHub Actions secrets, and the VSCode extension's `fetch()`.

## Files to Create

### `api/auth.py`
FastAPI `HTTPBearer` security dependency. If no key in config → passes through. If key present → validates credential or raises 401.

```python
from fastapi import HTTPException, Security, status
from fastapi.security import HTTPAuthorizationCredentials, HTTPBearer
import core.config as cfg

_bearer = HTTPBearer(auto_error=False)

def verify_api_key(
    credentials: HTTPAuthorizationCredentials | None = Security(_bearer),
) -> None:
    expected = cfg.get_api_key()
    if expected is None:
        return  # no key configured — open access
    if credentials is None or credentials.credentials != expected:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Invalid or missing API key",
            headers={"WWW-Authenticate": "Bearer"},
        )
```

### `cli/commands/api_key_cmd.py`
Typer sub-app with three commands:
- `devmemory api-key generate` — creates `secrets.token_hex(32)` (64 chars), saves to config, prints key once
- `devmemory api-key show` — prints the currently configured key (or warns none set)
- `devmemory api-key revoke` — removes key from config (re-opens the server)

## Files to Modify

### `core/config.py`
Add three helpers under a new `[api]` section (follows existing `[schedule]` / `[git]` pattern):
```python
def get_api_key() -> str | None:
    return load().get("api", {}).get("key") or None

def set_api_key(key: str) -> None:
    data = load()
    data.setdefault("api", {})["key"] = key
    save(data)

def delete_api_key() -> None:
    data = load()
    data.get("api", {}).pop("key", None)
    save(data)
```

### `api/server.py`
- Import `verify_api_key` and `Depends` from FastAPI
- Accept `auth_enabled: bool = True` parameter in `start_server()`
- When `auth_enabled`, apply `dependencies=[Depends(verify_api_key)]` to each `include_router()` call

### `cli/commands/serve.py`
Add `--no-auth` flag and pass to `start_server()`:
```python
no_auth: bool = typer.Option(False, "--no-auth", help="Disable API key enforcement")
```

### `cli/main.py`
Register the new sub-app:
```python
from cli.commands.api_key_cmd import app as api_key_app
app.add_typer(api_key_app, name="api-key")
```

### 4B.2 API Key Authentication

**Problem:** The REST API is unauthenticated by default. Safe for localhost, but
required before exposing to remote machines or CI/CD pipelines.

**Files:**
- `api/auth.py` — `verify_api_key` FastAPI dependency
- `cli/commands/api_key_cmd.py` — `devmemory api-key generate/show/revoke`
- `core/config.py` — `get_api_key()`, `set_api_key()`, `delete_api_key()`
- `api/server.py` — `Depends(verify_api_key)` on all routers + `--no-auth` flag
- `cli/commands/serve.py` — `--no-auth` option

**Behaviour:**
- No key in config → all requests accepted (safe for localhost default)
- Key in config → `Authorization: Bearer <key>` required on every request
- Wrong or missing key → `401 Unauthorized`

**Usage:**
```bash
devmemory api-key generate      # saves key, prints it once
devmemory serve                 # enforces auth if key configured
devmemory serve --no-auth       # disables enforcement (debug / trusted LAN)

curl -H "Authorization: Bearer <key>" "http://remote:7711/memory/search?q=redis"
```

## Verification Steps

```bash
# 1. Generate a key
devmemory api-key generate
# Expected: "API key saved. Bearer <key>"

# 2. Start server
devmemory serve &

# 3. Unauthenticated request → 401
curl -s "http://localhost:7711/memory/search?q=test"
# Expected: {"detail": "Invalid or missing API key"}

# 4. Authenticated request → results
curl -s -H "Authorization: Bearer <key>" "http://localhost:7711/memory/search?q=test"
# Expected: {"query": "test", "count": ..., "results": [...]}

# 5. --no-auth bypasses enforcement
devmemory serve --no-auth &
curl -s "http://localhost:7711/memory/search?q=test"
# Expected: results (no header needed)

# 6. Revoke key → server re-opens
devmemory api-key revoke
# Restart server — requests succeed without header

# 7. Run tests
uv run pytest api/tests/ -v
```

## Remote Testing Guide

How to verify the API works correctly from a second machine on the same network
or over the internet.

### Prerequisites on the server machine

```bash
# 1. Find the server's local IP
ipconfig getifaddr en0        # Wi-Fi
ipconfig getifaddr en1        # Ethernet
# e.g. 192.168.1.42

# 2. Generate a key (if not already done)
devmemory api-key generate
# Save the printed key — you'll need it on the client

# 3. Start the server bound to all interfaces (not just localhost)
devmemory serve --host 0.0.0.0 --port 7711
```

> **Firewall note:** macOS may prompt to allow incoming connections when you
> bind to `0.0.0.0`. Click Allow, or run:
> `sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add $(which uvicorn)`

---

### Step 1 — Verify connectivity from the client machine

```bash
# Replace 192.168.1.42 with the server's actual IP
ping 192.168.1.42

# Check the port is reachable
nc -zv 192.168.1.42 7711
# Expected: Connection to 192.168.1.42 7711 port [tcp/*] succeeded!
```

---

### Step 2 — Health check (no auth required)

```bash
curl -s http://192.168.1.42:7711/docs | head -5
# Expected: HTML for the FastAPI Swagger UI — confirms server is up
```

---

### Step 3 — Unauthenticated request should be rejected

```bash
curl -s "http://192.168.1.42:7711/memory/search?q=test"
# Expected: {"detail":"Invalid or missing API key"}
```

---

### Step 4 — Authenticated search

```bash
export DEVMEMORY_URL=http://192.168.1.42:7711
export DEVMEMORY_KEY=<paste key from server>

curl -s \
  -H "Authorization: Bearer $DEVMEMORY_KEY" \
  "$DEVMEMORY_URL/memory/search?q=authentication"
# Expected: {"query": "authentication", "count": ..., "results": [...]}
```

---

### Step 5 — Authenticated memory push (webhook)

```bash
curl -s -X POST "$DEVMEMORY_URL/memory/ingest" \
  -H "Authorization: Bearer $DEVMEMORY_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "text": "Remote test memory pushed from client machine",
    "source": "remote-test",
    "memory_type": "agent_solution",
    "tags": ["remote", "test"]
  }'
# Expected: {"status": "ok", "id": "..."}
```

Verify it landed on the server:

```bash
curl -s \
  -H "Authorization: Bearer $DEVMEMORY_KEY" \
  "$DEVMEMORY_URL/memory/search?q=remote+test+memory"
# Expected: result with summary "Remote test memory pushed from client machine"
```

---

### Step 6 — Wrong key returns 401

```bash
curl -s \
  -H "Authorization: Bearer wrongkey" \
  "$DEVMEMORY_URL/memory/search?q=test"
# Expected: {"detail":"Invalid or missing API key"}
```

### Exposing over the internet (optional)

For access from outside your LAN (CI/CD, GitHub Actions, remote machine):

**Option A — ngrok (quickest):**
```bash
# On server machine
ngrok http 7711
# ngrok prints a public URL like https://abc123.ngrok.io
# Use that URL instead of the local IP on the client
```

**Option B — SSH tunnel from the client:**
```bash
# On client — forward remote port 7711 to localhost:7711
ssh -L 7711:localhost:7711 user@server-host

# Then in another terminal on the client:
curl -s \
  -H "Authorization: Bearer $DEVMEMORY_KEY" \
  "http://localhost:7711/memory/search?q=test"
```

**Option C — GitHub Actions (CI/CD):**
```yaml
- name: Query DevMemoryIndex
  env:
    DEVMEMORY_URL: ${{ secrets.DEVMEMORY_URL }}
    DEVMEMORY_KEY: ${{ secrets.DEVMEMORY_KEY }}
  run: |
    curl -s \
      -H "Authorization: Bearer $DEVMEMORY_KEY" \
      "$DEVMEMORY_URL/memory/search?q=deploy+failure" | jq .
```
